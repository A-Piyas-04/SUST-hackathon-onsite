# Held-out validation evidence artifacts

Generated by the Phase 7 harness — never hand-edited.

```powershell
# from backend/
python -m app.scripts.validation_cli run
# or: make validate
```

| File | Source | Contents |
|---|---|---|
| `validation-summary.json` | `make validate` | Latest completed held-out run with every persisted metric (analytics + performance + reliability). |
| `performance-reliability.json` | `make validate` | Performance (latency) and reliability measurements only. |

Judge-facing interpretation: [`../validation-evidence.md`](../validation-evidence.md).

## Metric definitions (held-out split, frozen before measurement)

| Code | Category | Definition |
|---|---|---|
| `anomaly_precision` | analytics | TP / (TP + FP) over held-out (scenario × provider) anomaly cells; alertable = `requires_review` flag. |
| `anomaly_recall` | analytics | TP / (TP + FN) over held-out labelled anomaly cells. |
| `anomaly_false_positive_rate` | analytics | FP / (FP + TN) over held-out anomaly-negative cells (includes suppressed DQ case). |
| `shortage_lead_time_minutes` | analytics | `projected_shortage_at − as_of_at` for the held-out Scenario A shared-cash projection. |
| `data_quality_incident_rate` | reliability | Share of provider feed assessments classified stale/missing/conflicting across held-out runs. |
| `alert_explanation_coverage` | reliability | Published high-impact alerts with complete EN explanation sections / total published. |
| `api_avg_ms` | performance | In-process average latency over dashboard / liquidity / anomaly read handlers. |
| `api_p95_ms` | performance | In-process p95 latency for the same handlers. |

Honesty note: metrics are measured on a small synthetic held-out set with frozen seeds `2001`/`2002`/`2003`. Moderate-dataset CSV rows are fixture consistency checks, not harness output.
